## 概要
- **スリープ**: DeepSleepの確認できます。 DeepSleep の時間は３０秒で、起床後ワークカウンタが４０になるまでプログラムが実行される  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
プログラム実行中にSW2を押すことで、スリープを実行するかしないかを決められます
- **起動時の状態表示**: 何に起因して起動したかや例外発生で起動した時の情報をします
- **RTC RAM の使用**: RTC RAMに起動回数とスリープモードの記録をします
- **信号エッジ検出**: ボタンのＯＮ／ＯＦＦ時のエッジ検出し、ＯＮ時、ＯＦＦ時のフラグをセットする機能があります
- **ＡＤＣ**: ＭＣＵ内臓のアナログコンバーターを使う例があり、内部ＶＣＣかＡ０ポートを読むか切替えて使えます
- **ブザー**: ブザーにてスリープに入る時と、起床時におしらせ、ＳＷ２ボタンを押している間ブザーＯＮ
- **時間管理**: Tickerによる定周期割り込みと、TaskSchedulerによる非ブロッキングなイベント処理
- **タスク管理**: 割り込み内では実行トリガー（restart）のみを発行し、実処理をタスクで行う
- **低負荷**: タスク実行時以外は アイドル（disable）状態 とし、WiFi通信等のシステムリソースを最大限に確保

## 動作環境
- **ハードウェア**: 
  - IOT Integrated Controller V1　RRH-G101A REV-A (ESP8266搭載)
- **開発ツール**: PlatformIO (VSCode)
- **使用ライブラリ**: 
  - [TaskScheduler](https://github.com/arkhipenko/TaskScheduler)
  - Ticker (ESP8266 Core内蔵)

## 接続設定
- **JP1設定**: RST＃側にジャンパー（GPIO16 をリセットに接続します。GPIO16の初期化は不要）
- **JP5設定**: Buzzerジャンパー ＯＮ（GPIO13 をブザー制御にセット）
- **シリアル通信**: 115200 bps

## セットアップと書き込み
1. VSCodeでこのプロジェクトフォルダを開きます。
2. PlatformIOの `Upload and Monitor` を実行します。(※ブザーのジャンパーがＯＮの場合、書込み時にブザーが鳴ります。)
3. シリアルモニターにタスク実行ログ、ＡＤＣデータやボタン押下・離上情報などが表示されます。
